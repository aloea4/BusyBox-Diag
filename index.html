核心設計亮點

01. 零外部依賴與嚴格分層

採用「量測、分析、策略、展示」嚴格的四層架構設計。直接讀取 /proc 與 statfs()，不依賴 libprocps 或 ncurses，確保各層邏輯獨立。

02. Libdiag 共用基礎設施

獨立出 libdiag 模組統一處理字串解析與系統呼叫容錯。上層工具透過標準化結構體取用資料，實現真正的關注點分離。

03. 統一與輕量部署

深度依賴 BusyBox 的 argv[0] applet dispatch 機制，並支援 symlink 呼叫，達成單一執行檔多重功能的極致輕量化。

04. 智慧輸出與管線防呆

整合 isatty() 偵測終端機環境。支援 table、raw 與 json 輸出，一旦被 pipe 導出則自動關閉 ANSI 控制字元避免資料污染。

系統診斷工具集深潛

ptop

輕量行程資源分析器

專為批次分析與腳本化監控設計，徹底解決傳統互動式 top 的自動化痛點。

  • 精確時間點基準：採用 clock_gettime(CLOCK_MONOTONIC)，避免時間逆跳。
  • Snapshot-Delta 演算法：擷取雙時間點的 CPU jiffies 快照進行差分，還原真實使用率。
  • VmRSS 記憶體觀測：直接反映系統真實實體記憶體壓力，摒棄虛擬記憶體總量。
diagfs

檔案系統健康檢測工具

無縫整合空間與 Inode 監控，專注解決容器時代的掛載複雜性。

  • Namespace 感知：解析 /proc/self/mounts，突破 Docker 隔離環境視角誤差。
  • 防溢位與 Ceiling 策略：Inode 計算採無條件進位，確保臨界點前精準觸發警報。
diagnet

網路連線狀態監控器

被動式的核心 Socket 觀察器，不主動發送封包。

  • 常數級空間複雜度：串流回呼介面，純統計模式下記憶體開銷不再隨連線數量暴增。
  • Endianness 處理：精準將 Little-endian 十六進位還原為標準 Dotted-quad 格式。
