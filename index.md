
> **專為嵌入式環境打造的輕量整合式 Linux 系統診斷工具組**

---

## 📌 專題成果

 在資源受限的 Linux 嵌入式環境中，傳統的系統診斷工具（如 top、df、ss 等）彼此獨立，不僅需要安裝多套工具，且各自依賴不同的函式庫，導致部署成本過高且介面不一致。

本專題成功在單一的 BusyBox binary 中，整合了 Process（行程）、Filesystem（檔案系統）與 Network（網路）三大領域的完整診斷能力。系統底層共用統一的量測層（`libdiag`），並提供一致的輸出介面，在為 IoT 裝置、路由器韌體與容器環境提供低開銷、高整合的系統監控基礎設施。本系統的實作目標並非取代 GNU/Linux 全功能工具，而是在嵌入式診斷需求與 BusyBox 輕
量部署模型之使，提供一個兼顧實、性與資源效率的解決方案。

---

## 🌟 核心設計亮點

* **🚫 零外部依賴 (Zero External Dependencies)：** 直接讀取 Kernel 提供的虛擬檔案系統（`/proc`）與系統呼叫（`statfs()`）。完全捨棄 `libprocps`、`libnl` 或 `ncurses` 等龐大依賴，確保在最小化的 rootfs 中也能穩定執行。
* **🏗️ 共用量測層架構 (Shared Measurement Layer)：** 獨立出 `libdiag` 底層基礎設施，統一封裝 procfs 與 statfs 的解析與容錯邏輯。上層工具共享一致的資料結構，大幅降低重複實作風險與維護成本。
* **📦 統一與輕量部署 (Unified Applet Dispatch)：** 深度整合 BusyBox 的 `argv[0]` dispatch 路由機制。所有工具編譯至單一執行檔中，執行期記憶體佔用（Max RSS）極低，完美契合 IoT 與邊緣裝置的嚴苛限制。
* **🤖 自動化與管線友善 (Pipeline Ready)：** 內建 `isatty()` 終端機環境偵測。全面支援 `table`（含 ANSI 色彩的人機互動）、`raw`（適合 `grep`/`awk` 的管線處理）與 `json`（結構化數據）三種無縫切換的輸出格式。

## 🧰 系統診斷工具集

### 1. `ptop`：輕量行程資源分析器
專為自動化監控與腳本化採樣設計的行程監控器。
*  **Snapshot-Delta 架構：** 捨棄傳統  top 沉重的 ncurses 互動介面，透過雙時間點快照差分計算，精準還原 CPU 的真實使用率趨勢。
*  **穩健的配對機制：** 採用對前一份快照排序並以二元搜尋配對 PID 的策略，穩定處理 Linux 行程動態生成的生命週期。

### 2. `diagfs`：檔案系統健康檢測工具
無需依賴外部工具即可進行空間與 metadata 監控。
*  **深度元資料解析：** 直接呼叫 `statfs()` 提供空間使用、inode 狀態與 extent 分佈資訊。針對 inode 使用率採用 Ceiling（無條件進位）策略，避免因整數截斷而低估風險。
*  **Namespace 感知能力：** 透過解析 `/proc/self/mounts`，確保在 Docker 等隔離環境中，能精確反映當前容器視角下的掛載點狀態。

### 3. `diagnet`：網路連線狀態監控器
 被動式核心 socket 狀態觀察器，不介入核心排程、不主動發送封包。
*  **串流管線與 $O(1)$ 空間複雜度：** 在 `--stats` 純統計模式下，採用單次串流聚合設計，記憶體佔用退化為常數級別，完美適應嵌入式場景。
*  **啟發式安全觀察：** 內建策略層，能根據自訂規則為主機自動標記可疑的監聽埠。

## 🚀 效能與驗證成果

在針對嵌入式環境的嚴格標準下，BusyBox-Diag 展現了卓越的效能。在連續 200 次迭代的基準測試中，三項模組均成功達成 **「與系統原生工具效能差距小於 50%（比值 $\le 1.50$）」** 的專案要求，且在執行記憶體開銷上具備壓倒性優勢。

| 診斷模組 | 核心功能 | 對照基準工具 (Reference) | 效能比值 (Custom / Ref) | 測試結果 |
| :--- | :--- | :--- | :--- | :--- |
| **`ptop`** | 雙時間點行程資源快照 | GNU `top` | **0.868** *(更快速)* | ✅ 通過 |
| **`diagfs`** | 空間與 Inode 狀態採集 | GNU `df` (`df -k`, `df -i`) | **0.986** *(更快速)* | ✅ 通過 |
| **`diagnet`** | TCP/UDP 狀態串流解析 | iproute2 `ss` (`ss -tna`) | **1.196** *(符合標準)* | ✅ 通過 |

> *註：效能比值 = 自製工具執行中位數時間 / 對照工具執行中位數時間。數值越低代表效能越好。*
